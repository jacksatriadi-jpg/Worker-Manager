## RINGKASAN PROYEK

Worker Manager adalah web app berbasis Node.js (Express) + Vanilla JS
untuk mengelola worker (mesin/instance) dan pengguna (user).
Tema: Dark Glassmorphism (indigo/violet), serupa Dropbox bulk-signup UI.

## STRUKTUR FILE

Worker-Manager/
├── server.js          — Express server (API backend)
├── db.js              — JSON database module (CRUD users & workers)
├── data.json          — File database (dibuat otomatis, JANGAN commit)
├── package.json       — Dependencies Node.js
├── .gitignore
├── Agent.txt          — File ini
└── public/
    ├── index.html     — SPA (Single Page Application)
    └── index.css      — Semua styling (Dark Glassmorphism)

## CARA MENJALANKAN

1. Install dependencies:
   > npm install

2. Jalankan server:
   > npm start
   atau
   > node server.js

3. Buka browser: http://localhost:3000

## DATABASE (data.json)

Format JSON sederhana, berisi:
{
  "users": [
    {
      "id": "string (timestamp)",
      "username": "string",
      "password": "bcrypt hash",
      "role": "admin" | "user",
      "createdAt": "ISO string"
    }
  ],
  "workers": [
    {
      "id": "string (timestamp)",
      "name": "string",
      "workerLink": "string (URL)",
      "apiLink": "string (URL)",
      "status": "active" | "inactive",
      "createdAt": "ISO string",
      "updatedAt": "ISO string (optional)"
    }
  ]
}

PENTING: Hapus data.json jika ingin reset database ke kondisi awal
(hanya admin default yang ada).

## API ENDPOINTS

### Auth
POST   /api/login            — Login (body: {username, password})
POST   /api/logout           — Logout
GET    /api/me               — Get current session user

### Users (Admin only)
GET    /api/users            — Get all users
POST   /api/users            — Create user (body: {username, password, role})
DELETE /api/users/:id        — Delete user (tidak bisa hapus diri sendiri)

### Workers (Auth required)
GET    /api/workers          — Get all workers
POST   /api/workers          — Create worker (Admin) {name, workerLink, apiLink}
PATCH  /api/workers/:id/status — Update status {status: "active"|"inactive"}
DELETE /api/workers/:id      — Delete worker (Admin only)

## FRONTEND (SPA)

- Satu file index.html, tidak ada framework
- State disimpan di variabel JS: currentUser, workers[], users[]
- Session management via express-session (cookie)
- Tab "Pengaturan" hanya muncul jika role = admin
- Toast notification di pojok kanan bawah

## CATATAN TEKNIS

1. Database menggunakan JSON file (data.json), bukan SQL/NoSQL.
   Cocok untuk skala kecil. Jika perlu skala besar, migrasi ke SQLite
   (gunakan better-sqlite3) atau MongoDB.

2. Session secret ("worker-manager-secret-2024") harus diganti dengan
   variabel environment di production:
   process.env.SESSION_SECRET || 'fallback-secret'

3. bcrypt hash untuk "Bali@123" yang di-hardcode:
   $2a$10$SFubiddOHvxUtEmuZ1PC/OJr6zLUbNEHbhChQOTC7SzsPQRI9IkXC
   Jika hash tidak cocok, generate ulang dengan:
   const bcrypt = require('bcryptjs');
   console.log(bcrypt.hashSync('Bali@123', 10));
   Lalu update di db.js dan hapus data.json agar terinisialisasi ulang.

4. Port default: 3000. Bisa diubah via environment variable PORT.

5. Semua UI ada di public/index.html (vanilla JS, tidak pakai React/Vue).
   CSS ada di public/index.css (custom dark glassmorphism).
